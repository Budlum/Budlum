# Chapter 3.6: Optimistic QC and PQ Attestation

Optimistic QC lets Budlum keep normal operation fast while still preparing for post-quantum verification.

## 1. Why "Optimistic"?

The chain commits compact roots and verifies heavy post-quantum material only when needed. Honest operation remains light; suspicious data can still be challenged.

## 2. Data Structure: `QcBlob`

`QcBlob` groups validator attestations, post-quantum signatures, Merkle roots, and metadata for a checkpoint. The block can commit to the blob without carrying every large signature directly in the header.

## 3. PQ Fraud Proof

If a validator includes an invalid PQ signature:

1.  A challenger prepares the bad signature and its Merkle path.
2.  A `PqFraudProof` message is broadcast.
3.  The blockchain engine verifies the proof and slashes the validator that produced the fraudulent blob.

## 4. Advantages

1.  **Performance:** heavy post-quantum math does not burden the main path.
2.  **Light blocks:** large signatures do not bloat headers.
3.  **Future safety:** Budlum can later move from optimistic mode to strict verification.

## Summary

Optimistic QC balances today's performance with tomorrow's cryptographic requirements.
